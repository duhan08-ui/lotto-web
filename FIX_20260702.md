## 1. 검토에서 발견된 문제 수정

### ① 로컬 우선 렌더의 세션 상태 구멍 (app.py)
- `main()` 초기 동기화 블록에서 동기화 완료 후 `counts / unlock_granted /
  data_access_granted / simulation_count` 4개 키를 세션에 재주입.
- 첫 렌더에서 구버전 로컬 값으로 시드된 세션이 이후 `_persist_runtime_state`
  호출 시 원격 최신 값을 덮어쓰던 문제 방지.
- 재주입은 **원격 영속화(Supabase)가 켜진 경우에만** 수행
  (`log_utils.remote_persistence_enabled()` 신규 헬퍼 — 로컬/테스트 환경에서는
  의미가 없고 세션을 불필요하게 건드리므로 스킵).

### ② 캐시 메모리 상한 (app.py)
- 로그 토큰으로 키가 바뀌는 캐시들이 무제한 누적되어 장기 실행 시 RAM이
  증가하던 문제 예방. `max_entries` 적용:
  - `_load_enriched_history_cached`, `_prepare_history_analytics`,
    `_analyze_logs_cached`, `_recommend_freq_df`, `_read_excel_cached`,
    `_load_actual_result_map` → 3
  - `_log_full_csv_cached` → 10 (탭 5종 × 2세대)
  - `_get_cached_predictor` (cache_resource) → 2

## 2. UI 변경 2건

### ⑤ 조회 기간 카드 날짜 정렬 (app.py)
- `_format_period_panel_value()` 신규: '~' 없이 시작/종료일을 줄바꿈으로
  세로 정렬.
- 회차 모드: `이번주 1231회 로그` 제목 아래에 날짜 두 줄(작은 서브 톤).
- 날짜 모드: 시작일 아래 종료일.
- 캡션("선택 기간: ...")의 범위 표기는 기존 그대로 유지.

### ⑥ 캘린더 오늘 날짜 강조 (app.py)
- `_render_clickable_calendar`에 오늘 날짜 셀 전용 CSS 주입
  (`st-key-history_calendar_YYYY-MM-DD` 키 기반, 날짜선택 팝업의 '오늘'과
  동일한 틸 톤 링/배경, 선택(primary) 상태보다 우선).
- 오늘 셀 캡션을 `오늘 · N건`으로 표기.
