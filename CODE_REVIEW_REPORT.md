## 0. 가장 중요한 전제 (반드시 먼저)

> **"1등이 나오게" 코드를 수정하는 것은 수학적으로 불가능합니다.**
>
> 한국 로또(6/45)는 매 회차가 독립적이고 균등한 무작위 추첨이며, 어떤 6개 조합이든
> 1등 당첨 확률은 **1 / 8,145,060** 으로 동일합니다. 과거 출현 빈도, 마르코프 전이,
> 베이즈 추정, 안티패턴 필터링 등 **어떤 알고리즘도 다음 회차의 당첨 확률을 바꾸지
> 못합니다.** 지난 결과가 다음 추첨에 영향을 준다고 보는 것은 통계학의 *도박사의
> 오류(gambler's fallacy)* 입니다.
>
> 통계적으로 유효한 유일한 개입은 **공동당첨(분배금) 회피**입니다. 사람들이 많이
> 고르는 조합을 피하면 "만약 당첨될 경우" 1인당 수령액이 커집니다. 당첨 확률이
> 아니라 당첨 시 기대 수령액에만 영향을 줍니다.

이 보고서는 위 전제 위에서, **엔지니어링 품질**과 **수식/문서의 정확성**을 점검하고
실제로 의미 있는 수정을 적용한 내역을 담습니다.

## 2. 수정 완료 항목

### ✅ 2-1. Kelly 공식 → 공동당첨 위험도로 정직하게 재정의

**문제점**: 기존 `_kelly_criterion`은 켈리 공식 `f* = (b·p − q)/b`에 진짜 당첨확률이
아니라 "상대적 우열 점수의 시그모이드(`p_normalized`, 기준 0.5)"를 `p`로 대입했고,
`b = 0.45`도 켈리의 순배당률(net odds)이 아니라 기대수익률을 잘못 대입한 것이었습니다.
그 결과 `strong / avoid` 같은 등급이 산출되었지만, 모든 조합의 실제 기대값은 동일하게
음수이므로 **통계적 근거가 없는 오해 유발 신호**였습니다. (게다가 이 값들은 UI에서
전혀 사용되지 않는 죽은 코드였습니다.)

**조치**: `_jackpot_sharing_estimate()`로 교체. 당첨 확률을 계산하지 않고, 사람들이
선호하는 인기 패턴(생일대 1~31 쏠림, 연속수, 등차수열, 끝자리 반복, 5의 배수 쏠림,
홀짝/고저 완전 편중)의 포함 정도로 **공동당첨 위험도**를 0~1 점수와 낮음/보통/높음
등급으로 산출합니다.

```text
[1,2,3,4,5,6]        → sharing_grade=높음 (0.88)  ← 인기 패턴 다수
[3,14,27,33,41,44]   → sharing_grade=낮음 (0.0)   ← 분배 위험 낮음
```

반환 키 변경: `kelly_fraction / kelly_p_normalized / kelly_recommendation`
→ `sharing_risk_score / sharing_grade / sharing_reasons`

### ✅ 2-2. 문서-코드 수식 불일치 정리

`FORMULA_OPTIMIZATION_1223.md`가 실제 코드와 거의 전부 달랐습니다.

| 문서(기존) 주장 | 실제 코드 |
|----------------|-----------|
| Markov `prior_strength = 0.15` | 코드에 0.15 없음. gap은 `32.0`, state는 `18.0` |
| `P(gap)=(count+2)/(sum+90)` | 실제: `(success + (6/45)·32) / (total + 32)` |
| `score = base_score − volatility_penalty` | `volatility_penalty` 미존재. 실제는 앙상블 블렌딩 |
| 안티패턴 "3연속 0.4배 / 4연속 0.1배" | 실제: 고구간 개수 **선호 분포** (3)0.28 (4)0.50 (5)0.22 |

→ 문서를 **실제 코드 기준으로 전면 재작성**하고, 상단에 통계적 사전 고지를 추가했습니다.

- **secrets.toml 테스트가 실제 비밀 파일을 요구** (`.gitignore`와 모순): 안전한 템플릿

- **상태 기본값 테스트**: 코드가 의도적으로 '제한 모드'(unlock_granted=False) 기본으로

- **분석 로그 테스트**: 분석 요약 로그 저장이 의도적으로 제거됐으므로(달력 미표시),

## 4. "안티패턴"의 진짜 가치 (유일하게 통계적으로 유효)

당첨 확률은 못 올리지만, 인기 패턴 회피는 **당첨 시 분배금**을 높입니다. 한국 로또에서
1등이 여러 명이면 당첨금을 균등 분할하므로, 남들이 안 고르는 조합을 선택하면 (당첨이라는
희박한 사건이 일어났을 때) 독식 가능성이 커집니다. 현재 `anti_pattern_lotto.py`와 새
`_jackpot_sharing_estimate`는 이 목적에 실제로 부합합니다.
